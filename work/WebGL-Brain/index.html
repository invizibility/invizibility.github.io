WebGL Brain

2013
XTK.js, dat.gui.js, FreeSurfer
https://github.com/invizibility/fmribrain

Description

This project was a collaboration with the Waisman Brain Imaging Laboratory at UW-Madison. I reached out to Waisman wanting to learn about visualization problems in neuroscience my sophomore year of college, and was forwarded to Dave Perlman, a Waisman researcher working with pain stimulus fMRI data. Dave’s research involves, in his own words, “studying the time course of the brain’s response to pain as an indicator of ruminative or elabrorative mental processes that alter the overall aversive tone of the stimulus by extending the duration of affective response.” Visualizing the 4-dimensional data set of activation throughout the brain over a time interval was a cumbersome task, creating a bottleneck for researchers trying to garner insights about affective chronometry in the brain. At the time, visualization packages provided extremely crude 3D rendering and almost no useful animation for activation maps. I took on this project as a way to learn JavaScript and to become acquainted with biological imaging modalities, and to help Dave close the gap between his research results and an adequate presentation format. Over the course of a 2 semesters I became familiar with David’s data space and created a basic interactive JavaScript prototype.

Process

Before dealing with the visualization problem itself, I took on an introductory project of manually correcting algorithmically generated cortical surface files, so as to understand the data pipeline involved in neuroscience research. This step was immensely helpful for me, and understanding how different types of data that come off the MRI machine are computationally processed and analyzed really helped contextualize my design strategy. Next, I had to figure out the appropriate rendering context for the dataset. After understanding the diversity of data types I would have to visualize, I learned about XTK.js – a JavaScript API for parsing geometry from common medical imaging file formats.

Designing the brain viewing interface in JavaScript involved calibrating the registration and affine transformation of the anatomical volume, thresholding and layering the functional data over the anatomical data correctly, and assembling an intuitive GUI to bring it all together. Although I fell short of the ideal goal of having time series activation running with a slider in the browser, this project was effectively my first foray into computer graphics and helped me learn the fundamentals of WebGL. In addition to the programming exposure this project also gave me experience creating interactive prototypes for scientific visualization problems.
